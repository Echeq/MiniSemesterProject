## Overview

The `system_config` table exists in the database but has no frontend UI and no RPC functions for reading/writing. Profile preferences RPCs (`get_profile_preferences`, `set_profile_preferences`) exist but are not consumed by the frontend. This plan adds:

1. Backend: RPC functions to read/write `system_config`
2. Frontend: `useSystemConfig` hook
3. Frontend: "System Settings" tab in AdminModal
4. Integration: Apply settings on app load

## Phase 1 — Backend: RPC Functions

### Migration File

Create `supabase/migrations/20260702000000_system_config_rpc.sql`

### SQL Content

```sql
-- 1. RPC to get all system config as JSON object
create or replace function public.get_system_config()
returns jsonb
language plpgsql
security definer
as $$
begin
  return coalesce(
    jsonb_object_agg(key, value),
    '{}'::jsonb
  )
  from public.system_config;
end;
$$;

grant execute on function public.get_system_config() to authenticated;

-- 2. RPC to set a single config key-value
create or replace function public.set_system_config(
  p_key   text,
  p_value jsonb
)
returns void
language plpgsql
security definer
as $$
begin
  insert into public.system_config (key, value)
  values (p_key, p_value)
  on conflict (key)
  do update set
    value      = excluded.value,
    updated_at = now();
end;
$$;

grant execute on function public.set_system_config(text, jsonb) to authenticated;

-- 3. RPC to delete a system config key
create or replace function public.delete_system_config(p_key text)
returns void
language plpgsql
security definer
as $$
begin
  delete from public.system_config where key = p_key;
end;
$$;

grant execute on function public.delete_system_config(text) to authenticated;
```

## Phase 2 — Frontend: `useSystemConfig` Hook

### File

Create `frontend/src/hooks/useSystemConfig.js`

### Implementation

```jsx
import { useCallback, useEffect, useState } from 'react'
import { supabase } from '../api/supabaseClient'

const DEFAULT_CONFIG = {
  site_name: 'PivotPoint',
  default_language: 'en',
  default_theme: 'light',
  tasks_per_page: 50,
}

export function useSystemConfig() {
  const [config, setConfig] = useState(DEFAULT_CONFIG)
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState(null)

  const fetchConfig = useCallback(async () => {
    setLoading(true)
    setError(null)
    try {
      const { data, error: rpcError } = await supabase.rpc('get_system_config')
      if (rpcError) throw rpcError
      setConfig((prev) => ({ ...prev, ...(data || {}) }))
    } catch (err) {
      setError(err.message)
    } finally {
      setLoading(false)
    }
  }, [])

  const setConfigValue = useCallback(async (key, value) => {
    const { error: rpcError } = await supabase.rpc('set_system_config', {
      p_key: key,
      p_value: typeof value === 'string' ? JSON.stringify(value) : value,
    })
    if (rpcError) throw rpcError
    setConfig((prev) => ({ ...prev, [key]: value }))
  }, [])

  const deleteConfig = useCallback(async (key) => {
    const { error: rpcError } = await supabase.rpc('delete_system_config', { p_key: key })
    if (rpcError) throw rpcError
    setConfig((prev) => {
      const next = { ...prev }
      delete next[key]
      return next
    })
  }, [])

  useEffect(() => { fetchConfig() }, [fetchConfig])

  return { config, loading, error, fetchConfig, setConfigValue, deleteConfig }
}
```

### Step 3c: Create `SystemSettings` sub-component

Add inside `AdminModal.jsx` at the end of the file (after the existing sub-components):

```jsx
function SystemSettings({ onClose }) {
  const { t } = useTranslation()
  const { config, loading, error, setConfigValue, fetchConfig } = useSystemConfig()
  const [busy, setBusy] = useState(false)
  const [local, setLocal] = useState({})
  const [saved, setSaved] = useState(false)

  useEffect(() => {
    if (config) setLocal({ ...config })
  }, [config])

  async function handleSave() {
    setBusy(true)
    setSaved(false)
    try {
      const changed = Object.entries(local).filter(
        ([k, v]) => config[k] !== v,
      )
      for (const [key, value] of changed) {
        await setConfigValue(key, value)
      }
      setSaved(true)
      setTimeout(() => setSaved(false), 2000)
    } catch (err) {
      setError(err.message)
    } finally {
      setBusy(false)
    }
  }

  if (loading) {
    return <p className="py-8 text-center text-sm text-[var(--fg-muted)]">{t('admin.loading')}</p>
  }

  return (
    <div className="flex flex-col gap-4">
      {error && (
        <p className="rounded-md border px-3 py-2 text-sm" style={{ color: 'var(--danger)', borderColor: 'var(--danger)', background: 'var(--danger-soft)' }}>
          {error}
        </p>
      )}

      <label className="block">
        <span className="mb-1.5 block text-sm font-medium text-[var(--fg)]">Site Name</span>
        <input
          type="text"
          value={local.site_name || ''}
          onChange={(e) => setLocal((p) => ({ ...p, site_name: e.target.value }))}
          maxLength={100}
          className="input"
        />
      </label>

      <label className="block">
        <span className="mb-1.5 block text-sm font-medium text-[var(--fg)]">Default Language</span>
        <select
          value={local.default_language || 'en'}
          onChange={(e) => setLocal((p) => ({ ...p, default_language: e.target.value }))}
          className="input"
        >
          <option value="en">English</option>
          <option value="es">Español</option>
          <option value="zh">中文</option>
          <option value="id">Bahasa Indonesia</option>
          <option value="ar">العربية</option>
        </select>
      </label>

      <label className="block">
        <span className="mb-1.5 block text-sm font-medium text-[var(--fg)]">Default Theme</span>
        <select
          value={local.default_theme || 'light'}
          onChange={(e) => setLocal((p) => ({ ...p, default_theme: e.target.value }))}
          className="input"
        >
          <option value="light">Light</option>
          <option value="dark">Dark</option>
        </select>
      </label>

      <label className="block">
        <span className="mb-1.5 block text-sm font-medium text-[var(--fg)]">Tasks Per Page</span>
        <input
          type="number"
          min={10}
          max={200}
          step={10}
          value={local.tasks_per_page || 50}
          onChange={(e) => setLocal((p) => ({ ...p, tasks_per_page: parseInt(e.target.value, 10) || 50 }))}
          className="input"
        />
      </label>

      <div className="flex items-center justify-between pt-2">
        <div>
          {saved && (
            <span className="text-sm font-medium text-[var(--done)]">Settings saved ✓</span>
          )}
        </div>
        <div className="flex gap-2">
          <button type="button" onClick={onClose} className="btn btn-default">
            {t('project.cancel')}
          </button>
          <button type="button" onClick={handleSave} disabled={busy} className="btn btn-primary">
            {busy ? t('profile.saving') : t('project.save')}
          </button>
        </div>
      </div>
    </div>
  )
}
```

```jsx
import { useSystemConfig } from './hooks/useSystemConfig'

function App() {
  // ... existing code ...

  const { config, loading: configLoading } = useSystemConfig()

  // Apply default language
  useEffect(() => {
    if (config?.default_language) {
      i18n.changeLanguage(config.default_language)
    }
  }, [config?.default_language])

  // Apply default theme
  useEffect(() => {
    if (config?.default_theme && !localStorage.getItem('taskflow-theme')) {
      // Only apply if user hasn't manually set a theme
      toggleTheme(config.default_theme)
    }
  }, [config?.default_theme])
```

### Step 4b: Handle loading state

The `useSystemConfig` hook loads on mount. If it fails (no network, no admin), it falls back to `DEFAULT_CONFIG`. This is safe — the app still works without system config.

## Viability Assessment: M7 Plan

### Strengths

1. **Database table already exists** — `system_config` is fully created with proper RLS (admin-only). No schema migration needed for the table itself.
2. **Simple RPC pattern** — the get/set/delete RPCs follow the exact same pattern as existing RPCs in the codebase (e.g., `export_all_data`, `restore_from_backup`).
3. **No breaking changes** — the system config is optional. If the RPCs fail, the app uses `DEFAULT_CONFIG` and continues working.
4. **Small surface area** — only 1 new hook file, modifications to 1 existing component (AdminModal), and 1 integration point (App.jsx).
5. **Auth is already handled** — the `system_config` table has admin-only RLS; `security definer` on the RPCs ensures admin privileges.

### Risks

1. **Admin permission required** — if a non-admin calls `get_system_config`, the RPC is `security definer` so it returns data. But the frontend tab is only shown to admins. No security risk, but worth noting.
2. **Caching** — the hook fetches config on mount but doesn't poll. If another admin changes settings, other admins won't see the update until page reload. Acceptable for an admin panel.
3. **Language change UX** — changing the default language via System Settings won't immediately change the current user's language (it only applies on next login). To apply immediately, call `i18n.changeLanguage()` in the save handler.

### Dependencies

- `supabase` client must be available (always is)
- `i18n` must be initialized (it is, via `src/i18n.js`)
- AdminModal must receive `isAdmin` prop (it does, from App.jsx)

### Edge Cases

1. **Empty config** — if `system_config` table is empty, `get_system_config()` returns `{}`, and `DEFAULT_CONFIG` values are used.
2. **Partial config** — if only `site_name` is set, `default_language` and `default_theme` fall back to `DEFAULT_CONFIG`.
3. **Concurrent writes** — two admins saving simultaneously: last write wins. Acceptable for low-frequency admin operations.
4. **Invalid JSONB values** — the `set_system_config` RPC passes `p_value` as `jsonb`. Any string must be `JSON.stringify()`'d first. The hook handles this.
