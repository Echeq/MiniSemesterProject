#### Step 3: Add date formatting helper (15 min)

Create a small utility function:

```jsx
function formatDateForExport(dateStr, format = 'display') {
  if (!dateStr) return ''
  const d = new Date(dateStr + 'T00:00:00')
  if (isNaN(d.getTime())) return dateStr
  if (format === 'csv') return dateStr  // ISO for CSV
  if (format === 'excel') return d
  return d.toLocaleDateString(undefined, { year: 'numeric', month: 'short', day: 'numeric' })
}
```

      }
    })
  })
  downloadCSV([headers, ...rows], 'tasks.csv')
}

## Viability Assessment: M4 + M6 Plan

### Strengths

1. **Gantt and 3D Sphere are already implemented** — the task.md checked status is inaccurate. The actual code in `GanttView.jsx` and `DataSphere.jsx` already contains working dependency arrows and flowing particles. This means 2 out of 4 M4 sub-modules require only verification, not development.

2. **Bulk edit reuses existing primitives** — `updateTask`, `deleteTask`, and `useBoard` already handle single-task operations. No new backend work required.

3. **Export enhancement is additive** — the existing export functions (CSV, PDF, Excel) just need column filtering and date formatting added. The existing code structure supports easy extension.

4. **No database migrations needed** — all features are frontend-only (except the Gantt/Sphere verification which is also frontend).

### Risks

1. **Bulk edit performance** — calling `updateTask()` or `deleteTask()` sequentially for N tasks makes N individual API calls. If N > 20, this could be slow. Mitigation: show a loading indicator during batch operations; limit selection to 50 tasks with a warning.

2. **Export column picker UX** — the column picker dropdown position relative to the export dropdown could overlap or close unexpectedly. Mitigation: use `e.stopPropagation()` and position the picker above or to the side on mobile.

3. **Testing time** — the plan estimates 30 min for verification but real testing of Gantt drag-to-link across browsers may take longer.

### Dependencies

- `updateTask` and `deleteTask` must be available in ListView — currently passed as props to Board, which passes to ListView. Verify the prop chain.
- GanttView already receives `onAddDependency` / `onRemoveDependency` from App.jsx (line 376).
