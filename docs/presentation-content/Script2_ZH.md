### [幻灯片 3] 主题 1：问题分析 —（40 秒）

"传统的项目管理工具存在五个显著问题。

第一，**工具割裂**。大多数团队混用电子表格、邮件和基础任务追踪器，各系统之间互不连通。

第二，**缺乏实时协作**。你更新了一个任务，队友必须刷新页面才能看到变化。

第三，**配置复杂**。Jira 需要专用的服务器和复杂的配置，对大多数团队来说过于臃肿。

第四，**许可费用高昂**。按用户收费的模式对成长型团队来说成本快速攀升。

第五，**定制受限**。多数 SaaS 工具限制了工作流程的灵活性。

那么团队真正需要的是什么？轻量、实时、价格合理、可定制的工具。这正是 PivotPoint 要解决的问题。"

---

### [幻灯片 4] 主题 2：解决方案 —（35 秒）

"PivotPoint 是一个功能完整的看板系统，针对上述五个问题逐一给出了解决方案。

**拖拽看板** —— 核心界面。任务在待办、进行中和已完成三列间流转。

**实时同步** —— 所有连接客户端之间即时同步。一人操作，全员实时可见。

**基于角色的权限控制** —— 管理员、成员和未知用户三级权限，通过数据库行级安全策略实施。

**多视图模式** —— 不只有看板。甘特图做时间线规划，3D 球体做空间展示，仪表盘做统计概览，列表视图做数据表格。

**导出功能** —— 支持 Excel、PDF 和 CSV，可自定义选择导出列。

**国际化** —— 5 种语言支持，含阿拉伯语从右到左布局。"

### [幻灯片 5] 主题 3：业务流程 —（45 秒）

"让我们看看完整的用户流程，可以对照业务流图。

用户访问应用。首先系统检查 Supabase 环境变量是否已配置。如未配置，显示设置提示页面 —— 这对克隆项目的开发人员很重要。

如已配置，用户通过 Supabase Auth 认证 —— 登录或注册。

认证后，系统检查用户角色，有三种可能：

- **未知用户** —— 首次注册且无邀请的用户。看到空白看板，可点击请求访问按钮。经管理员批准前无法查看任何任务。

- **成员** —— 被管理员邀请或请求被批准的用户。可查看所有任务，但只能编辑分配给自己的任务。

- **管理员** —— 首个注册的用户自动成为管理员。对所有任务拥有完全控制权，可管理团队成员。

角色检查完成后，用户进入看板主工作区。"

### [幻灯片 6] 主题 4：系统架构 —（55 秒）

"现在介绍系统架构，可以对照架构图。

前端使用 **React 19** 和 **Vite 8**。约 **40 个 React 组件** 平铺在 `src/components/` 下。所有业务逻辑封装在 **10 个自定义 Hook** 中，例如 `useBoard` 处理任务操作、`useAuth` 处理认证、`useProjects` 处理项目 CRUD、`usePresence` 处理在线状态、`useTaskEditing` 显示谁在编辑什么。

前端有 **5 种视图模式**：看板、列表、甘特图、3D 球体和仪表盘，各自实现为独立组件。

后端方面，关键设计决策是：**没有自定义服务器**。整个后端采用 **Supabase**，它提供了一切所需：

- **Auth** —— 开箱即用的邮箱/密码认证
- **PostgREST API** —— 基于 PostgreSQL 模式自动生成的 REST API，含行级安全策略
- **Realtime** —— 数据库变更的 WebSocket 订阅
- **Storage** —— 文件存储（如头像上传）
- **PostgreSQL** —— 关系型数据库

数据库是单一数据源，通过 **29 个 SQL 迁移文件** 在 `supabase/migrations/` 中管理。每次模式变更都通过迁移实现 —— 生产环境不允许直接 SQL 修改。

前后端通过 `supabase-js` 客户端库通信，自动处理认证令牌、REST 调用和实时订阅。"

### [幻灯片 7] 主题 5：任务管理 —（55 秒）

"任务管理是 PivotPoint 的核心。让我逐一介绍本页展示的功能。

**CRUD 操作。** 用户可创建、读取、更新和删除任务。创建表单包含标题、描述、截止日期、状态、优先级、负责人和项目 —— 全部在一个模态框中完成。

**拖拽功能。** 基于 @dnd-kit 实现。任务可在列内拖拽排序，也可跨列拖拽改变状态。桌面端使用 PointerSensor（6px 激活距离），移动端使用 TouchSensor（200ms 延迟防误触）。

**优先级系统。** 四级：P0 紧急（红色）、P1 高（橙色）、P2 中（黄色）、P3 低（绿色）。每级在任务卡片上视觉区别明显。

**截止日期与智能视图。** 任务可设置截止日期。三种智能视图自动筛选：『我的任务』显示分配给你的任务，『即将到期』显示 7 天内到期的任务，『已逾期』显示过期的任务。

**任务分配。** 每项任务可分配给项目成员。负责人的头像会显示在任务卡片上，便于快速识别。

**任务依赖。** 任务可依赖其他任务。当有阻塞任务时，该任务无法移动到「完成」列 —— 系统在移动前会调用数据库 RPC 检查。阻塞数量显示在任务卡片上。

**标签。** 每个项目可自定义带颜色的标签，通过 LabelManager 组件管理，可在任务上开关。

**排序系统。** 每个任务有一个 float8 类型的 position 字段。重排时取相邻任务的中点值，插入时取最大值加 1024。这样避免了耗时的重新索引操作。"

### [幻灯片 8] 主题 6：实时协作 —（50 秒）

"这是我们的核心功能之一。让我对照数据流图解释实时同步的工作原理。

当用户 A 将一个任务移动到「完成」列时，具体流程如下：

**第一步 —— 前端。** `useBoard` Hook 调用 `supabase.from('tasks').update({ status: 'done' }).eq('id', taskId)`。

**第二步 —— 后端。** PostgreSQL 处理 UPDATE。行级安全策略验证用户权限。变更提交到数据库。

**第三步 —— Realtime 广播。** 由于 `tasks` 表已加入 `supabase_realtime` 发布，Supabase Realtime 捕获变更并通过 `board` 频道广播。

**第四步 —— 所有客户端。** 用户 B、用户 C 及其他所有连接客户端收到变更事件。每个浏览器中的 `useBoard` Hook 都维护了一个订阅：

```javascript
.channel('board')
.on('postgres_changes',
  { event: '*', schema: 'public', table: 'tasks' },
  (payload) => { /* 合并到状态 */ }
)
.subscribe()
```

**第五步 —— 合并。** Hook 使用 requestAnimationFrame 批量处理事件，按任务 ID 去重，重新获取修改行的完整数据（含标签、依赖、负责人信息），合并到本地状态。整个过程在一秒内完成。

**除了任务更新外**，我们还有：

- **在线状态追踪** —— `usePresence` Hook 订阅 `presence` 频道。每个用户广播自己的用户 ID 和显示名。在线状态自动在所有客户端间同步。侧边栏中在线成员旁显示绿点。

- **编辑指示器** —— `useTaskEditing` Hook 追踪每个用户正在编辑的任务。其他用户会在任务卡片上看到头像和『编辑中…』提示。

- **通知** —— `tasks` 表上的数据库触发器会向 `notifications` 表插入记录，用于截止日期提醒、逾期通知和新任务分配。`useNotifications` Hook 通过 Realtime 订阅获取这些通知。"

### [幻灯片 9] 主题 7：视图与导出 —（35 秒）

"PivotPoint 提供五种视图。让我逐一介绍：

**看板** —— 默认视图。待办、进行中、已完成三列，支持拖拽和列显隐切换。移动端一次显示一列，支持滑动切换。

**甘特图** —— 基于 frappe-gantt 构建。带截止日期的任务以时间轴上的条形显示。支持从四分之一天到月多级缩放。用户可拖拽条形调整日期。任务间可绘制依赖箭头，点击箭头可删除依赖。

**3D 球体** —— 基于 Three.js 构建。每个任务是一个球体表面的粒子。完成率决定球体颜色从红到绿渐变。进行中的任务以琥珀色粒子轨道环绕显示。逾期任务触发红色线框脉冲动画。用户可拖拽旋转、滚轮缩放。

**仪表盘** —— 个人概览页面。显示任务总数、已完成、进行中和逾期统计卡片。进度环显示完成百分比。团队在线状态和最近任务也在该页面展示。

**列表视图** —— 可排序表格。支持列排序、状态/优先级/标签筛选、日期范围筛选、紧凑模式和批量编辑（复选框批量改状态和删除）。

**导出功能。** 用户可将当前筛选视图导出为 Excel（基于 exceljs）、PDF（基于 jsPDF）或 CSV。导出模态框可选择列、文件名和格式。"

### [幻灯片 10] 主题 8：技术栈 —（30 秒）

"快速过一下技术选型。

**前端：** React 19，Vite 8 构建工具（生产环境使用 Rolldown Rust 打包器）。Tailwind CSS v4 样式，广泛使用 CSS 自定义属性实现主题切换（明亮/暗黑）。@dnd-kit 拖拽库。

**后端：** 纯 Supabase。无自定义 Express 或 NestJS 服务器。Supabase 提供 Auth、PostgREST API、Realtime WebSocket 和 Storage。

**数据库：** PostgreSQL，29 个 SQL 迁移文件。每张表都有行级安全策略。数据库触发器处理通知和验证。

**测试：** Vitest 4 + Testing Library + jsdom。116 个单元测试加 API 集成测试。

**CI/CD：** GitHub Actions —— 每次推送到主分支或 PR 时自动运行测试和构建。

**国际化：** i18next，5 种语言文件：英语、西班牙语、中文、印度尼西亚语、阿拉伯语。"

### [幻灯片 11] 数据库设计 —（35 秒）

"让我们来看数据库结构，对照 ER 图。

**7 个核心表：**

- **profiles** —— 关联 auth.users，存储显示名、头像 URL 和角色
- **tasks** —— 主表，含标题、描述、状态、优先级、负责人、截止日期、position、项目 ID
- **projects** —— 项目分组，含名称、描述、颜色、图标和状态
- **labels** 和 **task_labels** —— 多对多关系，用于任务分类
- **task_dependencies** —— 任务间的阻塞关系
- **notifications** —— 用户通知（截止日期、分配、导出）

辅助表：**invitations**、**join_requests**、**project_members** 用于团队管理。

关键约束：标题 1-200 字符，描述最多 5000 字符，显示名最多 100 字符（触发器自动截断）。任务的 `created_by` 字段在插入后不可修改 —— 通过列级授权确保。`position` 字段为 float8 类型。"

---

### [幻灯片 12] 基于角色的权限控制 —（40 秒）

"权限控制通过三种角色配合 PostgreSQL 行级安全策略实现。可以对照 RBAC 图。

**管理员角色：** 首个注册用户通过数据库触发器自动成为管理员。管理员对所有项目和共享看板的任务拥有完全 CRUD 权限。可管理团队成员、邀请用户、提升权限、归档删除项目、访问管理面板。

**成员角色：** 被管理员邀请或请求被批准的用户。成员可查看项目和共享看板中的所有任务，但只能编辑分配给自己的任务。不能创建、删除或更改任务分配。移动端底部弹出框显示『查看』按钮而非『编辑』，桌面端点击任务打开只读详情卡片。

**未知用户角色：** 无邀请的首次注册用户由 `handle_new_user()` 触发器分配为『unknown』角色。他们看到完全空白的看板，只能点击请求访问按钮。提交请求后，管理员在管理面板的 Access 标签页中处理。

**注册流程：** `handle_new_user()` 数据库触发器在每次 Auth 注册时触发。检查是否有待处理的邀请（匹配邮箱）。如有，用户成为 member。如无，成为 unknown。首次注册绕过此逻辑直接成为 admin。"

### [幻灯片 13] 开发流程 —（20 秒）

"开发流程保持简洁：

- 在 `frontend/` 目录下执行 `npm install`
- 复制 `.env.example` 为 `.env`，填入 Supabase 凭据
- `npm run dev` 启动 Vite 开发服务器（localhost:5173）
- `npm test` 运行 116 个单元测试
- `npx vitest run tests/api.test.js` 运行 API 集成测试
- `supabase start` 启动本地 PostgreSQL
- `supabase db push` 应用迁移
- `npm run build` 使用 Rolldown 进行生产构建"

### [幻灯片 15] 现场演示 —（60 秒）

"现在让我演示 PivotPoint 的实际操作。

首先，我登录系统。注意这是一个新用户 —— 因为是首次注册，将自动成为管理员。

我创建一个名为『Sprint 24』的项目，添加描述，选择颜色和图标。

接着，我创建几个任务，设置不同的优先级、截止日期和负责人。

注意看 —— 我将一个任务从待办拖到进行中 —— 状态立即更新。现在我在列内重新排序任务。

我给一个任务分配负责人、添加标签、设置依赖关系。

现在切换到甘特图 —— 任务以时间轴上的条形显示。我放大视图，拖拽条形调整截止日期。

切换到 3D 球体 —— 每个粒子代表一个任务。可以看到从红到绿的渐变基于完成率。悬停显示任务详情。

打开导出模态框，选择列，选择 Excel 格式，下载。

最后，我打开第二个浏览器窗口演示实时同步。看 —— 我在这里移动一个任务，另一个窗口即时更新。在线指示器显示两个用户都已连接。

这就是 PivotPoint。"
